## Add

- `detect --ocr` — recognize the actual text inside `detect` (additive on the broad default), so `detect img --ocr --lang zh-Hans,en-US` returns everything in the image including the words. `detect` previously gave text-region boxes only, never the content.
- Agent-friendly `--help`: the help output now renders a SYNOPSIS, a DESCRIPTION with TIP/NOTE blocks, and a TLDR section (modeled on `x eget --help`), so AI agents can operate macvision from its help alone. Populated for every subcommand. Cross-cutting tips (JSON `ok` field, top-left pixel bbox convention, input sources) live on the root help.

## Note

- `detect` flag semantics clarified and documented in its help: a flag like `--faces` NARROWS to that single detector (faster, focused); `--rects` and `--ocr` are ADDITIVE on top of the broad default.
- Bounding boxes remain pixel `[x, y, w, h]` with a top-left origin, plus a normalized `norm` copy.
- Daemon `detect` request now also accepts `ocr` (bool) and `lang` ([string]).
