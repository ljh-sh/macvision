## Add

- **Auto multi-language OCR.** `macvision ocr img.png` now reads a broad set of scripts (Latin, simplified/traditional Chinese, Japanese, Korean, …) in a single pass instead of English-only. Vision is natively multi-language, so the broad default adds only ~10–30 ms.
- **Repeatable `--lang`.** Specify multiple languages by repeating the flag, and order matters for CJK: `macvision ocr img.png --lang ja-JP --lang en-US` puts the Japanese recognizer first (it handles kanji + kana), Latin second. Comma form (`--lang ja-JP,en-US`) still works.
- **Named language presets** via `--lang`: `all` (broad default), `cjk`, `cn`, `latin`, `european`, `en`.
- **Custom presets.** Define your own with an environment variable, then reference it: `MACVISION_LANG_SEA="th-TH,vi-VN,id-ID" macvision ocr img.png --lang sea`. Useful for agents that want a domain-specific broad set.
- **Aggregate `confidence`** at the top level of the OCR result (mean of the top-candidate confidences). Lets an agent judge quality and retry with a different language set — `count: 0` means the language didn't match the script.
- **`testdata/`** — rendered English / Chinese / Japanese / mixed images plus a `render.swift` generator, used by new integration tests.

## Note

- This release also ships the `--version` / `-V` flag and automatic SLSA build-provenance generation that were staged as v0.1.2 but never tagged. v0.1.3 is the first release since v0.1.1; `macvision --version` reports `macvision 0.1.3`.
- CJK ordering: Vision commits to the **first** CJK language's recognizer for the whole line and does not fall back across CJK. Chinese first yields zero results on Japanese (no kana support). Put the script-matching language first, or let an agent retry (`zh` → `ja` → `ko`) using `count` + `confidence` as the signal.
